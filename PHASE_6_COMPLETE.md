## Функционал

### Автоматическое логирование

#### Типы действий
- **Команды**: `command:start`, `command:help`, `command:ls`, и т.д.
- **Сообщения**: `message` (обычные текстовые сообщения)
- **Загрузки**: `file_upload`, `photo_upload`, `voice_upload`

#### Что логируется
- User ID
- Тип действия (action)
- Детали (первые 100 символов текста, имя файла, и т.д.)
- Успех/неудача
- Timestamp

#### Пример записи
```json
{
  "user_id": 1,
  "action": "command:ls",
  "details": "/app/data",
  "success": true,
  "created_at": "2026-05-03 13:10:00"
}
```

### Просмотр логов

#### Для пользователей
- Свои последние 10 действий
- Своя статистика
- Нет доступа к логам других пользователей

#### Для администраторов
- Все логи всех пользователей
- Статистика по всем действиям
- Поиск по логам
- Просмотр логов конкретного пользователя

### Статистика

#### Пользовательская статистика
- Общее количество действий
- Количество успешных действий
- Количество неудачных действий
- Дата первого действия
- Дата последнего действия

#### Глобальная статистика (admin)
- Топ действий по частоте
- Успешные/неудачные для каждого действия
- Общая картина использования бота

### Поиск

#### Возможности
- Поиск по названию действия
- Поиск по деталям
- Case-insensitive (ILIKE)
- До 20 результатов

#### Примеры
- `/audit_search file` - все действия с файлами
- `/audit_search error` - все ошибки
- `/audit_search api_key` - все действия с API ключами

## Архитектура

### Слои

```
Middleware (audit.py)
    ↓
Service Layer (audit_service.py)
    ↓
Database (audit_logs table)
```

### Поток данных

1. **Пользователь отправляет сообщение**
2. **AuthMiddleware** - проверяет пользователя
3. **AuditMiddleware** - логирует действие
4. **Handler** - обрабатывает команду
5. **AuditMiddleware** - обновляет статус (success/failure)
6. **AuditService** - сохраняет в БД

### База данных

#### Таблица audit_logs
```sql
CREATE TABLE audit_logs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    action VARCHAR(255) NOT NULL,
    details TEXT,
    success BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

#### Индексы
- `idx_audit_logs_user_id` - для фильтрации по пользователю
- `idx_audit_logs_created_at` - для сортировки по времени
- `idx_audit_logs_success` - для фильтрации по статусу
- `idx_audit_logs_action` - для фильтрации по действию

## Безопасность

### Доступ к данным

#### Пользователи
- Видят только свои логи
- Видят только свою статистику
- Нет доступа к логам других пользователей

#### Администраторы
- Видят все логи
- Видят статистику всех пользователей
- Могут искать по всем логам
- Могут просматривать логи конкретного пользователя

### Конфиденциальность

#### Что НЕ логируется
- Пароли
- API ключи (только факт добавления/удаления)
- Полное содержимое файлов
- Личные данные

#### Что логируется
- Типы действий
- Имена файлов
- Первые 100 символов текста
- Результат (успех/неудача)

## Производительность

### Оптимизация

- Асинхронное логирование
- Индексы на часто используемых полях
- Пагинация для больших выборок
- Graceful error handling (не блокирует основной поток)

### Масштабирование

- Партиционирование по дате (для больших объемов)
- Архивирование старых логов
- Агрегация статистики

### Метрики

- Количество действий в час
- Процент неудачных действий

- Самые активные пользователи

### Алерты

- Высокий процент неудачных действий
- Подозрительная активность
- Необычные паттерны использования
