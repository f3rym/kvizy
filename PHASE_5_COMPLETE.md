# Claude Telegram Bot - Phase 5: Simple Monitoring

## Что создано

### Сервисы

#### MonitorService (`src/services/monitor_service.py`)
- ✅ `get_system_stats()` - получение статистики системы
  - CPU: процент использования, количество ядер
  - Memory: использование, процент, общий объем
  - Disk: использование, процент, общий объем
  - Network: отправлено/получено данных
  - Uptime: время работы системы
- ✅ `get_docker_stats()` - статистика Docker контейнеров
  - Список контейнеров
  - CPU и Memory для каждого контейнера
  - Статус контейнеров
- ✅ `check_alerts()` - проверка превышения порогов
  - Автоматическая проверка метрик
  - Генерация предупреждений
- ✅ `set_threshold()` - установка порогов предупреждений
- ✅ `get_thresholds()` - получение текущих порогов

### Обработчики команд

#### Мониторинг (все пользователи)
- ✅ `/stats` - полная статистика системы
  - CPU, Memory, Disk, Network
  - Uptime
  - Автоматические предупреждения
- ✅ `/docker` - статистика Docker контейнеров
  - Список всех контейнеров
  - CPU и Memory каждого контейнера
  - Статус (running/stopped)
- ✅ `/health` - быстрая проверка здоровья
  - Краткая сводка
  - Только критичные метрики
  - Предупреждения если есть
- ✅ `/alerts` - показать текущие пороги предупреждений

#### Администрирование (только админы)
- ✅ `/set_alert <metric> <value>` - установить порог предупреждения
  - Метрики: cpu_percent, memory_percent, disk_percent
  - Значение: 0-100

## Функционал

### Системная статистика

#### CPU
- Процент использования (с интервалом 1 секунда)
- Количество ядер
- Предупреждение при превышении порога (по умолчанию 80%)

#### Memory
- Общий объем (GB)
- Использовано (GB)
- Процент использования
- Предупреждение при превышении порога (по умолчанию 85%)

#### Disk
- Общий объем (GB)
- Использовано (GB)
- Процент использования
- Предупреждение при превышении порога (по умолчанию 90%)

#### Network
- Отправлено данных (MB)
- Получено данных (MB)
- Суммарная статистика с момента загрузки

#### Uptime
- Время работы системы
- Форматирование: HH:MM:SS

### Docker мониторинг

#### Контейнеры
- Список всех контейнеров
- Статус каждого контейнера
- CPU использование (%)
- Memory использование (MB и %)

### Система предупреждений

#### Пороги по умолчанию
- CPU: 80%
- Memory: 85%
- Disk: 90%

#### Настройка порогов
- Только для администраторов
- Динамическое изменение без перезапуска
- Валидация значений (0-100)

#### Автоматические предупреждения
- Проверка при каждом запросе `/stats`
- Отображение в `/health`
- Логирование в системный лог

## Как использовать

### 1. Проверить статистику системы

```
/stats

📊 Статистика системы

⏱ Uptime: 2:15:30

🖥 CPU:
• Использование: 45.2%
• Ядер: 4

💾 Память:
• Использовано: 2.5 GB / 8.0 GB
• Процент: 31.2%

💿 Диск:
• Использовано: 15.3 GB / 50.0 GB
• Процент: 30.6%

🌐 Сеть:
• Отправлено: 1250.5 MB
• Получено: 3420.8 MB
```

### 2. Проверить Docker контейнеры

```

✅ claude-telegram-bot
• Статус: running

✅ postgres
```

### 3. Быстрая проверка здоровья

```

CPU: 45.2%
Memory: 31.2%
Disk: 30.6%
```

### 4. Проверить пороги предупреждений

```

• CPU: 80.0%
• Memory: 85.0%
• Disk: 90.0%

Используйте /set_alert для изменения порогов.
```

### 5. Установить новый порог (только админ)

```
/set_alert cpu_percent 70

✅ Порог cpu_percent установлен на 70.0%
```

## Предупреждения

### Когда срабатывают

Предупреждения автоматически отображаются в `/stats` и `/health` когда метрики превышают пороги:

```
/stats

📊 Статистика системы
...

⚠️ Предупреждения:
• ⚠️ CPU usage high: 85.3%
• ⚠️ Memory usage high: 87.1%
```

### Логирование

Все предупреждения логируются:
```
WARNING - Alerts triggered: ['⚠️ CPU usage high: 85.3%']
```

## Архитектура

### Компоненты

```
Handler Layer (monitor.py)
    ↓
Service Layer (monitor_service.py)
    ↓
psutil / docker
```

### Библиотеки

#### psutil
- Кроссплатформенная библиотека для системной информации
- CPU, Memory, Disk, Network, Process мониторинг
- Легковесная и быстрая

#### docker (опционально)
- Python SDK для Docker API
- Статистика контейнеров
- Graceful fallback если Docker недоступен

## Безопасность

### Доступ к командам

#### Все пользователи
- `/stats` - просмотр статистики
- `/docker` - просмотр контейнеров
- `/health` - проверка здоровья
- `/alerts` - просмотр порогов

#### Только администраторы
- `/set_alert` - изменение порогов

### Ограничения

- Нет доступа к процессам других пользователей
- Только чтение системной информации
- Нет возможности управления контейнерами

## Производительность

### Оптимизация

- CPU процент с интервалом 1 секунда (баланс точности/скорости)
- Кэширование не требуется (данные всегда актуальные)
- Легковесные операции (< 100ms)

### Нагрузка

- Минимальное влияние на систему
- Асинхронные операции
- Нет постоянного мониторинга (только по запросу)
